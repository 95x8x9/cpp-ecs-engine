# ECS 게임 엔진 구현 및 이를 응용한 군집 행동 시뮬레이션 시스템

# 프로젝트 주제 이름

SafeCrowd: ECS 게임 엔진 기반 군중 행동 시뮬레이션 및 의사 결정 지원 시스템

# 대상 이해당사자 (stakeholder)

이 프로젝트의 주요 이해당사자는 건물 관리자, 시설 운영자, 행사장 운영자, 안전관리자, 건축·공간 설계자, 재난 대응 담당자, 그리고 실제 공간을 이용하는 방문객 및 시민이다.

특히 건물이나 행사장에서는 평상시 이동 동선과 비상시 대피 동선이 복잡하게 얽혀 있어, 운영자와 관리자 입장에서 혼잡 구간이나 병목 구간을 사전에 예측하는 것이 중요하다. 따라서 설계 과정에서는 많은 사람이 동시에 이동할 때 어떤 문제가 발생하는지, 어떤 배치와 안내 방식이 더 안전하고 효율적인지를 이해해야 한다.

또한 전문 시뮬레이션 개발자가 아니더라도 시설 관리자나 운영자가 직접 자신의 건물 구조를 불러와 다양한 상황을 시험해볼 수 있는 사용자 중심 기능을 제공하고자 한다. 이를 통해 실제 현장 담당자도 별도의 프로그래밍 지식 없이 시뮬레이션을 활용할 수 있도록 하는 것을 목표로 한다.

# 이해당사자의 고충(pain point) 또는 니즈(needs)

이해당사자들은 실제 상황이 발생하기 전에는 군중의 움직임을 정확히 예측하기 어렵다는 문제를 겪는다. 건물 구조, 출입구 수, 통로 폭, 장애물 배치, 인원 규모에 따라 결과가 크게 달라지기 때문이다.

주요 고충과 니즈는 다음과 같다.

- 많은 인원이 동시에 이동할 때 병목 구간이 어디에서 발생하는지 미리 파악하기 어렵다.
- 실제 대피 훈련은 시간과 비용이 많이 들고, 반복적인 실험도 쉽지 않다.
- 행사장이나 건물 구조를 바꾸었을 때 그 변화가 안전성에 어떤 영향을 주는지 빠르게 검토하기 어렵다.
- 의사결정이 경험이나 직관에 의존하는 경우가 많아 객관적인 근거가 부족하다.
- 다양한 시나리오를 비교하여 더 안전하고 효율적인 동선 설계를 하고 싶다.
- 사용자가 자신이 관리하는 건물이나 행사장 구조를 직접 반영하여 시뮬레이션하고 싶어도, 이를 쉽게 입력하거나 불러올 수 있는 도구가 부족하다.
- 비상상황, 출구 폐쇄, 특정 구역 혼잡, 인원 증가 등 다양한 시나리오를 손쉽게 바꾸어 가며 비교할 수 있는 직관적인 UI가 필요하다.
- 전문가의 도움 없이도 건물 정보 입력, 시나리오 설정, 결과 확인까지 수행할 수 있는 사용성 높은 시스템이 필요하다.

# 이해당사자의 이유

이해당사자들은 군중 밀집으로 인한 안전사고를 줄이고, 비상 상황에서 더 빠르고 안정적으로 대피할 수 있는 환경을 만들고자 한다. 특히 건물 관리자나 행사 운영자는 사고 예방이 곧 인명 보호와 직결되기 때문에 이 문제 해결을 중요하게 여긴다.

또한 설계자와 운영자는 단순히 이론적으로 괜찮아 보이는 배치가 아니라, 실제 사람들의 움직임을 반영한 근거 기반 의사결정을 원한다. 이를 통해 통로 폭 조정, 출구 추가, 안내 표지 배치, 인원 분산 전략과 같은 대안을 비교하고 더 나은 선택을 할 수 있다.

특히 각 건물과 행사장의 구조, 운영 방식, 수용 인원이 서로 다르기 때문에, 사용자는 자신의 현장 조건에 맞는 맞춤형 시뮬레이션 결과를 직접 확인하고 싶어 한다.

# 프로젝트 수행자의 의도

게임 엔진 기술을 이용하여  실제 사회 문제 해결에 활용하려는 시도이다. 특히 ECS(Entity-Component-System) 구조는 많은 개체를 효율적으로 관리하는 데 강점이 있어, 다수의 사람을 동시에 시뮬레이션해야 하는 군중 모델링에 적합하다고 판단하였다.

건물이나 행사장에서의 이동과 대피 문제는 현실성과 공공성이 높은 주제이므로, 기술적 도전과 사회적 가치를 함께 담을 수 있는 프로젝트라고 생각하였다.

# 탐구 내용 및 기대 결과

프로젝트 기간 동안 ECS 기반 게임 엔진의 핵심 구조를 직접 설계·구현하고, 이를 바탕으로 다수의 군중 에이전트가 이동하는 시뮬레이션 시스템을 개발하고자 한다. 각 에이전트는 목표 지점으로 이동하면서 주변 개체와의 충돌을 피하고, 장애물을 우회하며, 특정 상황에서는 출구를 향해 대피하도록 설계할 예정이다.

또한 사용자가 직접 건물 또는 행사장 정보를 불러오고, UI를 통해 인원 규모, 출입구 상태, 장애물 배치, 위험 구역 설정 등의 조건을 변경하며 여러 시나리오를 실험할 수 있는 기능도 함께 구현하고자 한다. 이를 통해 사용자는 동일한 공간에서도 다양한 조건을 적용하여 결과를 비교하고, 보다 합리적인 의사결정을 내릴 수 있을 것으로 기대한다.

주요 탐구 내용은 다음과 같다.

- ECS 구조를 적용한 엔진 코어 설계 및 구현
- 다수 에이전트의 이동 처리 및 성능 최적화
- 군중 행동 모델(분리, 정렬, 회피, 목표 지향 이동 등) 적용
- 건물·행사장 구조를 반영한 맵 또는 공간 모델링
- 사용자가 건물 구조 데이터를 불러오거나 직접 설정할 수 있는 입력 기능 설계
- 다양한 시나리오를 UI에서 생성·수정·저장할 수 있는 인터페이스 구현
- 출구 폐쇄, 인원 밀집, 특정 구역 통제, 장애물 추가 등 상황 변화 시뮬레이션 기능 구현
- 평상시 이동 시나리오와 비상 대피 시나리오 비교
- 시나리오별 결과를 비교하여 의사결정에 활용할 수 있는 시각화 기능 구현
- 시뮬레이션 결과를 시각적으로 제시하고, 의사결정에 필요한 지표를 제공하는 기능 구현

기대 결과물은 다음과 같다.

- ECS 기반 군중 시뮬레이션 실행 프로그램(프로토타입)
- 건물 또는 행사장 환경을 반영한 테스트 시나리오
- 사용자가 자신의 건물 또는 공간 정보를 반영해 시뮬레이션할 수 있는 UI 기반 프로토타입
- 혼잡도, 병목 구간, 평균 이동 시간, 전체 대피 시간 등의 결과 지표
- 시나리오별 결과를 비교·분석할 수 있는 사용자 인터페이스
- 시나리오별 결과 비교 보고서 또는 발표 자료

정량적·정성적 달성 지표는 다음과 같다.

- 1,000명 이상의 에이전트를 안정적으로 처리할 수 있는 시뮬레이션 구현
- 최소 3가지 이상의 공간·대피 시나리오 비교 수행
- 사용자가 UI만으로 최소 3가지 이상의 시나리오를 직접 생성·변경·실행할 수 있는 기능 구현
- 전체 대피 시간, 구간별 밀집도, 출구별 사용률 등의 수치 제공
- 사용자 테스트에서 건물 구조 반영과 시나리오 설정이 직관적이라는 정성적 평가 확보
- 사용자 또는 지도교사 피드백을 통해 위험 구간 파악에 도움이 된다는 정성적 평가 확보
